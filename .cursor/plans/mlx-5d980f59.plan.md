# Port FIBO text encoder from PyTorch to MLX

## Goals

- Implement an MLX version of the FIBO text encoder (SmolLM3-like) that reproduces the PyTorch pipeline behavior.
- Load text-encoder weights into MLX (likely via safetensors) and keep tokenizer in PyTorch/Transformers for now.
- Preserve advanced prompt handling (negative prompts, padding, attention masks, guidance) and be LoRA-ready.

## High-level Steps

1. **Study the PyTorch text encoder behavior**

- Review `BriaFiboPipeline.get_prompt_embeds` and `encode_prompt` in `pipeline_bria_fibo.py` to fully capture:
- How prompts/negative prompts are tokenized and padded.
- How hidden states are combined into `prompt_embeds` and `prompt_layers`.
- How `text_ids` and attention masks are shaped and used.
- Inspect `SmolLM3ForCausalLM` config (hidden size, num layers, rotary, etc.) to understand the architecture to mirror in MLX.

2. **Design an MLX SmolLM3-style text encoder module**

- Create a new MLX module (e.g. `FIBOTextEncoderMLX`) that:
- Accepts `input_ids` and `attention_mask` tensors with the same shapes as in PyTorch.
- Implements embeddings, attention, MLP, rotary/positional encoding compatible with `SmolLM3ForCausalLM`.
- Returns `hidden_states` as a tuple/list so we can reconstruct `prompt_embeds` and `prompt_layers` exactly.
- Mirror the forward API we actually need (no need to support logits/language modeling head initially).

3. **Implement a tokenizer + bridge from PyTorch to MLX**

- Keep using `AutoTokenizer` from Transformers (CPU / PyTorch side) to generate `input_ids` and `attention_mask`.
- Convert those tensors to numpy and then to MLX arrays for the MLX text encoder.
- Reproduce the same empty-prompt handling (`bot_token_id`, mask fixes) as in PyTorch.

4. **Implement weight export and MLX weight loading**

- Add a small conversion script that:
- Loads the PyTorch `SmolLM3ForCausalLM` weights.
- Maps them into the MLX text-encoder parameter structure (following the patterns already used for Flux/Qwen).
- Saves them as MLX-friendly safetensors or `.npz` files.
- Add an MLX weight handler for FIBO text encoder that:
- Attempts to load pre-exported MLX weights first.
- Falls back (if needed) to on-the-fly PyTorch→MLX conversion using a declarative mapping.

5. **Recreate `encode_prompt` behavior in MLX**

- Implement an MLX-side `encode_prompt` helper that:
- Calls the tokenizer bridge, runs the MLX text encoder, obtains all hidden layers.
- Builds `prompt_embeds` as concat of the last two layers and repeats over `num_images_per_prompt`.
- Builds `prompt_layers` list (per-layer hidden states), `prompt_attention_mask`, and `text_ids` with matching shapes.
- Handles negative prompts and guidance scale in the same way as PyTorch (including padding via a `pad_embedding` equivalent).
- Ensure the outputs match the PyTorch A/B checkpoints numerically for a few test prompts.

6. **Wire the MLX text encoder into the FIBO MLX pipeline**

- Update the MLX `FIBO` pipeline to:
- Call the new MLX `encode_prompt` rather than relying on PyTorch-saved tensors via `debug_load`.
- Feed `prompt_embeds`, `prompt_layers`, `text_ids`, and attention masks directly into the MLX transformer, mirroring the PyTorch call site.
- Keep a debug mode where we can still fall back to loading PyTorch debug tensors for cross-checking.

7. **LoRA behavior for MLX (v1)**

- Explicitly skip dynamic LoRA support for the MLX text encoder in the first version.
- Assume any LoRA effects you care about are already baked into the exported base weights before conversion.
- Keep the MLX text encoder design modular enough that LoRA-style adapters could be added in a later iteration if needed.

8. **Validation and A/B tests**

- For a fixed prompt/negative prompt, run the PyTorch FIBO pipeline and log:
- `prompt_embeds`, `prompt_layers`, `text_ids`, and attention masks via `debug_save`.
- Run the MLX text encoder + `encode_prompt` on the same prompt and compare tensors elementwise.
- Once aligned, re-run the full FIBO A/B flow to confirm end-to-end behavior remains consistent.
